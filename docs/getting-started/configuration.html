⚙️ 配置详解

EchoSRT 的所有运行时配置集中在 config/config.json 文件中，由前端全局设置面板和后端配置服务共同管理。配置文件在首次启动时会自动从 config.example.json 复制生成，你可以在 Web UI 的"全局设置"面板中可视化编辑，也可以直接修改 JSON 文件。

配置文件结构总览​

{
  "system_settings": { /* 系统全局设置 */ },
  "secrets": { /* 密钥信息 */ },
  "model_settings": { /* 本地 Whisper 模型设置 */ },
  "transcribe_settings": { /* 本地识别参数 */ },
  "vad_settings": { /* VAD 语音活动检测 */ },
  "ffmpeg_settings": { /* FFmpeg 提取参数 */ },
  "llm_settings": { /* LLM 翻译多方案设置 */ },
  "online_asr_settings": { /* 云端 API 识别多方案设置 */ },
  "library": { /* 媒体库扫描设置 */ }
}

1. system_settings — 系统全局设置​

参数类型默认值说明
network_proxystring""全局网络代理地址，支持 http:// / https:// / socks5:// 协议。留空表示直连。例如："http://127.0.0.1:7890"
enable_global_proxybooleanfalse全局代理总开关。开启后会自动设置 HTTP_PROXY / HTTPS_PROXY 环境变量，影响所有网络请求。
use_proxy_for_model_downloadbooleanfalse是否将代理应用于 Hugging Face 模型下载。需要同时开启 enable_global_proxy 并填写 network_proxy。

SOCKS5 代理说明​

当配置 SOCKS5 代理时，系统会自动将 socks5:// 转换为 socks5h://，让代理服务器执行 DNS 解析，防止本地 DNS 污染阻断 Hugging Face 访问。

⚠️ 注意：全局代理总开关关闭后，即使各模块独立勾选了代理，也不会生效。

2. secrets — 密钥信息​

参数类型默认值说明
hf_tokenstring""Hugging Face 访问令牌（非必需）。用于下载需要认证的模型或提高 API 速率限制。

3. model_settings — 本地 Whisper 模型设置​

参数类型默认值说明
model_sizestring"large-v2"Whisper 模型规模。可选值由 faster_whisper.available_models() 动态提供。
download_rootstring"models"模型缓存目录（相对于项目根目录）。

4. transcribe_settings — 本地识别参数​

这些参数直接传递给 faster_whisper.WhisperModel.transcribe() 方法。

参数类型默认值说明
enginestring"local"识别引擎类型："local"（本地 Whisper）或 "api"（云端 API）。
languagestring/nullnull指定源语言代码。null 表示自动检测。
taskstring"transcribe"任务类型："transcribe"（转录原文）或 "translate"（翻译为英文）。
beam_sizeint5Beam Search 宽度。
best_ofint5采样候选数。
vad_filterbooltrue是否启用 VAD 过滤。

5. vad_settings — VAD 语音活动检测​

参数类型默认值说明
vad_filterbooltrue是否启用 VAD 过滤。检测并移除静音段，减少幻觉。
vad_parametersobject/nullnullSilero VAD 的细粒度参数。

6. ffmpeg_settings — FFmpeg 提取参数​

参数类型默认值说明
audio_trackstring"0:a:0"音频流选择器。
start_timestring""起始时间（HH:MM:SS）。
end_timestring""结束时间（HH:MM:SS）。

7. llm_settings — LLM 翻译设置 (v1.1.0 多方案系统)​

参数类型默认值说明
active_profile_idstring"default"当前激活的方案 ID。
profilesarray[...]方案列表。每个方案包含 id, name, api_key, base_url, model_name, concurrent_workers 等信息。
target_languagestring"zh"目标翻译语言代码。
use_network_proxyboolfalse是否为翻译请求单独启用代理。

Profile 方案结构​

每个方案对象包含以下字段：

  • id: 方案唯一标识符。
  • name: 方案显示名称。
  • api_key: LLM API Key。
  • base_url: API 基础 URL。
  • model_name: 模型名称。
  • batch_size: 每批发送的字幕条数。
  • concurrent_workers: 并发 Worker 数。
  • system_prompt: 自定义系统提示词。

8. online_asr_settings — 云端 API 识别设置 (v1.1.0 多方案系统)​

参数类型默认值说明
`active_profile_id"string"default"当前激活的方案 ID。
profilesarray[...]方案列表。每个方案包含 id, name, api_key, base_url, model_name 等信息。
languagestring/nullnull源语言代码（ISO 639-1）。
use_network_proxyboolfalse是否为云端 API 请求单独启用代理。

9. library — 媒体库扫描设置 (v1.1.0 新增)​

参数类型默认值说明
library_pathsarray[]扫描的本地或 NAS 文件夹路径列表。
allowed_extensionsarray[...]允许识别的后缀。默认包含 20+ 种音视频格式。
auto_scan_enabledbooleanfalse是否开启自动扫描。

支持的语言代码​

EchoSRT 支持 99 种语言的自动检测与识别。完整列表见 api/services/config_service.py 中的 SUPPORTED_LANGUAGES 字典，或在前端语言下拉框中查看。

配置持久化与热更新​

保存机制​

  • 前端修改任一配置项后，会触发 updateConfig() API 调用。
  • 后端使用增量合并策略保护 library 等关键配置不被意外抹除。
  • 保存成功后，后端立即调用 set_global_proxy() 动态生效代理设置。

恢复默认​

如果配置被错误修改导致服务异常，可以通过前端"恢复默认"按钮一键还原。建议在升级 v1.1.0 前备份您的 config.json 文件。
