# REST API 参考

EchoSRT 所有 HTTP 接口基于 FastAPI 构建，前端通过 `api.js` 中的封装函数调用。

---

## 配置 API

### GET `/api/config`
获取当前完整配置。

**响应**: `{ system_settings, transcribe_settings, llm_settings, audio_settings }`

### POST `/api/config`
更新配置。

**请求体**: 完整或部分 config JSON

### POST `/api/config/restore`
重置为 `config.example.json` 的默认配置。

---

## 任务 API

### POST `/api/upload/{asset_type}`
上传文件（video / audio / srt）。

**路径参数**: `asset_type` = `video` | `audio` | `srt`
**表单字段**: `file` (UploadFile), `task_id` (可选 str)

**响应**: `{ task_id, filename, message }`

### POST `/api/task/execute`
触发任务执行。

**请求体**:
```json
{
  "task_id": "abc123",
  "steps": ["extract", "transcribe", "translate"],
  "ffmpeg_settings": { ... },
  "transcribe_settings": { ... },
  "model_settings": { ... },
  "vad_settings": { ... },
  "llm_settings": { ... },
  "online_asr_settings": { ... },
  "system_settings": { ... }
}
```

**响应**: `{ "task_id": "abc123", "message": "工作流已加入流水线队列" }`

### GET `/api/download/{task_id}?type=original`
下载产物文件。

**查询参数**: `type` = `original` | `translated` | `audio` | `video`

**响应**: `FileResponse` (SRT, WAV 或 视频文件)

### GET `/api/task/{task_id}/assets`
获取单个任务的资产信息（包含就绪状态、基础名称、是否有视频等）。

### GET `/api/tasks`
列出所有任务列表。

**响应**: `[{ task_id, base_name, has_video, has_audio, has_original_srt, has_translated_srt, created_at, sort_weight }]`

### DELETE `/api/task/{task_id}`
删除单个任务的工作目录及其所有关联状态。

### DELETE `/api/task/{task_id}/asset/{asset_type}`
删除任务下的特定资产。`asset_type` 可选：`video`, `audio`, `original`, `translated`。

### POST `/api/tasks/reorder`
批量调整任务在前端工作区的显示排序。

**请求体**: `["task_id_1", "task_id_2", ...]` (按显示顺序排列的 ID 列表)

## 模型 API

### GET `/api/models`
获取本地可用的 Whisper 模型列表及下载状态。

### DELETE `/api/models/{model_id}`
删除已下载的本地模型文件。

### POST `/api/models/{model_id}/download`
触发后台异步下载指定模型。

**请求体**: `{ "system_settings": { ... } }` (用于传递代理设置)

### GET `/api/models/download_status`
获取当前所有正在进行的模型下载任务进度。

---

## LLM / ASR 模型发现

### GET `/api/llm/models?api_key=...&base_url=...`
从指定的 LLM 提供商拉取可用模型列表（OpenAI 兼容接口）。

### GET `/api/asr/models?api_key=...&base_url=...`
从指定的 ASR 提供商拉取可用模型列表。

---

## 系统

### GET `/api/system/info`
获取后端系统信息（设备类型 CPU/CUDA、GPU 名称等）。

### GET `/api/languages`
获取支持的语言字典代码与名称映射。

### POST `/api/proxy/test`
测试指定的代理地址是否可连通。

**请求体**: `{ "proxy_url": "http://127.0.0.1:7890" }`

---

## WebSocket

### WS `/ws/progress/{task_id}`
订阅实时任务进度推送。

### GET `/api/task/{task_id}/status`
(回退接口) 获取任务最新的 WebSocket 状态快照。

### GET `/api/pipeline/status`
(回退接口) 获取全局所有任务的步骤状态字典。
