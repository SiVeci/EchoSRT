# 🌍 LLM 翻译

EchoSRT 集成 LLM（大语言模型）实现字幕翻译，支持任意 OpenAI 兼容的 Chat Completions API，将 `original.srt` 翻译为目标语言并输出 `translated.srt`。翻译引擎支持异步并发批量处理，可在保持字幕时间轴完整的前提下高效翻译长视频字幕。

## 翻译流程详解

### 第一阶段：SRT 解析

每个元素是一个完整的 SRT 块（序号 + 时间轴 + 文本）。系统会自动处理换行符并清洗 DeepSeek R1 等模型的思考标签。

### 第二阶段：分批与上下文注入

系统会将字幕分批发送给大模型，并自动注入上一批的结尾作为上下文参考，以确保翻译的连贯性。

### 第三阶段：异步并发翻译

通过 `Semaphore` 严格控制并发数，防止触发 API 限流。

## 配置参数 (v1.1.0 多方案系统)

在 **🤖 LLM 翻译** 标签页中，您可以通过顶部的 **“切换方案”** 下拉框快速在不同的 API 配置之间切换。

### 核心参数

| 参数 | 说明 |
|------|------|
| `API Base URL` | 兼容 OpenAI 格式的接口地址。 |
| `API Key` | 您的 API 访问密钥。 |
| `Model Name` | 模型名称（如 gpt-4o, deepseek-chat）。支持点击刷新图标自动拉取。 |
| `翻译批次大小` | 每次发给模型的字幕行数。建议 30-80。 |
| `并发请求数` | 同时进行的翻译请求数。注意不要超过 API 提供商的限制。 |

### 多方案管理

点击方案选择器右侧的“设置”按钮，可以执行以下操作：
- **新增方案**：克隆当前配置。
- **重命名**：为当前配置取一个直观的名字。
- **删除**：移除不再使用的配置。

## 兼容的 LLM 服务

任何实现 `/v1/chat/completions` 端点的服务均可使用：
- OpenAI 官方
- DeepSeek
- SiliconFlow
- Groq / OpenRouter
- 本地自建 (Ollama / vLLM)

## 最佳实践

1. **翻译质量**：优先选择强模型（如 DeepSeek-V3），批次大小设为 50 左右。
2. **翻译速度**：对于低延迟服务（如 Groq），可适当增加并发数至 5-8。
3. **格式修复**：如果发现模型输出了多余的 Markdown 代码块，系统会自动进行清洗。

## 故障排除

- **429 报错**：触发了 API 限流，请减小并发请求数。
- **401 报错**：API Key 无效或过期。
- **500 报错**：服务商端异常，请尝试切换至备用方案。
