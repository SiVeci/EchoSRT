# 📁 任务工作区

任务工作区是 EchoSRT 的核心管理枢纽，负责视频导入、任务调度、状态监视及产物下载。v1.1.0 版本对扫描引擎和资产管理进行了大幅增强。

---

## 视频导入方式

### 1. 拖拽/手动上传
在任务列表上方区域，直接拖入视频或音频文件。系统将自动为其创建独立的 UUID 任务文件夹，并对音频执行 16kHz 重采样。

### 2. 媒体库自动扫描 (v1.1.0 增强)
您可以在“全局设置”中添加本地或 NAS 路径。
- **高性能扫描**：基于单次全目录遍历算法，秒级识别数千个文件。
- **广泛兼容**：支持 .mp4, .mkv, .wmv, .ts, .mp3, .m4a 等 20 余种主流格式。
- **原地处理**：媒体库导入的文件不会被复制到工作区，而是建立软链接引用，节省磁盘空间。

---

## 资产管理与自愈

### 磁盘状态自愈
v1.1.0 引入了“以磁盘为准”的资产识别逻辑。
- **无感复活**：如果您在工作区删除了一个来自媒体库的任务，该视频文件在下次“点击扫描”时会立即重新出现在“新发现”列表中。
- **状态同步**：系统会实时穿透磁盘 `meta.json` 检查任务指纹，确保不会出现重复导入。

### 单一资产清理
点击任务行的 **“资产包”** 标签，可以单独删除某个环节的产物（如删除 Wav 以释放空间）。
- **保护机制**：对于媒体库映射的视频原文件，系统会禁止单独删除，以防止误伤您的原始素材库。

---

## 任务监视与下载

### 实时日志与焦点
点击任务行的 **“监视”** 按钮，右侧控制台会立即切换至该任务。
- **长文件名感知 (v1.2.0)**：页面顶部横幅将显示当前正在处理的**视频源文件名**。遇到超长文件名时自动截断并提供悬浮提示，确保界面排版美观。
- **WebSocket 追踪**：即使关闭网页后再打开，只要任务在跑，连接后即可实时找回进度。

### 任务中断与清理 (v1.2.0)
当任务卡死或不再需要时，系统支持全链路中断机制：
- **一键中断**：支持对单任务或全部任务发送中断信号。
- **资源释放**：强制终止底层的处理进程，并自动清理中途产生的不完整临时文件，立即释放系统资源。

### 统一的高级下载 (v1.1.0)
点击“下载原声”或“下载翻译”标签：
- **精准命名**：下载的文件名将严格遵循 `{原文件名}_{语言代码}.srt`。
- **进度可见**：内置浏览器级别的下载进度条，并实时显示 MB/s 下载速度。

---
*提示：工作区目录位于项目根目录下的 `/workspace` 文件夹，建议定期清理不再需要的旧任务。*
