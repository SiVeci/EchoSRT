# ☁️ 云端 API 语音识别

EchoSRT 支持通过 OpenAI 兼容的 Audio API 进行云端语音识别（ASR）和语音翻译。你可以使用 OpenAI 官方服务，也可以接入任何兼容 `/v1/audio/transcriptions` 和 `/v1/audio/translations` 端点的第三方 API（如 Groq、DeepInfra、本地 Ollama 等）。

## 配置参数

在 **🔑 Whisper API** 标签页或 `config.json` 中配置：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `api_key` | string | `""` | API 密钥（必填） |
| `base_url` | string | `"https://api.openai.com/v1"` | API 基础 URL |
| `model_name` | string | `"whisper-1"` | 模型名称 |
| `language` | string | `""` | 音频语言代码。`""` 表示自动检测 |
| `prompt` | string | `""` | 提示词，引导模型识别特定词汇 |
| `translate` | bool | `false` | 开启后调用 translations 端点，直接输出英文翻译 |
| `speaker_labels` | bool | `false` | 开启说话人标注（需 API 支持） |
| `word_timestamps` | bool | `false` | 开启词级时间戳 |
| `use_network_proxy` | bool | `false` | 是否使用代理连接 API |

### 超时设置

在 `timeout_settings` 中可精细控制连接/读取超时：

```json
{
  "asr_config": {
    "timeout_settings": {
      "connect": 15.0,
      "read": 300.0
    }
  }
}
```

| 参数 | 默认值 | 下限 | 说明 |
|------|--------|------|------|
| `connect` | `15.0` | 3.0 秒 | TCP 握手超时 |
| `read` | `300.0` | 30.0 秒 | 等待 API 响应的超时。大文件可能需要更长时间 |

> ⚠️ `write` 超时固定为 60 秒，`pool` 超时固定为 10 秒。

## 大文件切片机制

当音频超过 **25 MB** 时，系统自动使用 FFmpeg 进行物理切片以绕过 API 文件大小限制并防止内存溢出：

### 切片策略

```python
CHUNK_LIMIT_MB = 24.0           # 触发切片阈值 (>24MB)
chunk_length_sec = 5 * 60       # 每片 5 分钟
```

```python
subprocess.run([
    ffmpeg_cmd, "-y", "-i", audio_path,
    "-f", "segment", "-segment_time", str(chunk_length_sec),
    "-ac", "1", "-ar", "16000",
    segment_pattern
], stdout=subprocess.DEVNULL, stderr=subprocess.PIPE, text=True)
```

切片文件命名模式：`echo_srt_temp_chunk_{uuid}_000.wav`, `_001.wav`, ...

### 防时间轴雪崩

切片后每个分片独立上传 API，返回的 SRT 时间码从 `00:00:00` 开始。必须累加偏移量进行全局对齐：

```python
# 核心：使用 ffprobe/WAVE header 读取切片的 "真实物理时长"
def get_audio_duration(file_path, fallback_duration):
    # 方法 1: 解析 WAV 头部
    with wave.open(file_path, 'rb') as wav_file:
        return wav_file.getnframes() / wav_file.getframerate()

    # 方法 2: ffprobe 精确读取
    # ffprobe -v error -show_entries format=duration ...

    # 方法 3: 回退到理论值 (5 * 60 = 300 秒)
    return fallback_duration
```

```python
current_offset = 0.0
for idx, chunk in enumerate(audio_chunks):
    offset_seconds = current_offset

    # 先识别当前切片...
    chunk_srt = api_call(chunk)

    # 然后将 SRT 时间码加上偏移量，重新编号
    shifted_srt, current_srt_index = parse_and_shift_srt(
        chunk_srt, offset_seconds, current_srt_index
    )
    final_srt_content += shifted_srt + "\n\n"

    # 关键：用真实物理时长而不是理论值，防止误差累积
    current_offset += get_audio_duration(chunk, fallback_duration=chunk_length_ms / 1000.0)
```

> 🧊 **为什么叫"防雪崩"**：如果每片只按理论 300 秒偏移，实际时长 ±0.5 秒的误差会随切片数 N 线性累积，导致第 N 片的字幕完全错位。使用真实物理时长可以彻底消除此问题。

## 响应格式

根据参数组合自动选择：

```python
use_verbose_json = speaker_labels or word_timestamps
req_format = "verbose_json" if use_verbose_json else "srt"
```

| 请求格式 | 条件 | 说明 |
|----------|------|------|
| `srt` | 默认 | API 直接返回 SRT 文本，无需后处理 |
| `verbose_json` | 开启说话人/词级时间戳 | 返回完整 JSON，需 `verbose_json_to_srt()` 手动拼接 |

### verbose_json 转 SRT

```python
def verbose_json_to_srt(response_data, use_words=False):
    items = data.get("words", []) if use_words and data.get("words") else data.get("segments", [])

    for i, item in enumerate(items, start=1):
        start = item.get("start", 0.0)
        end = item.get("end", 0.0)
        text = item.get("word", item.get("text", "")).strip()

        # 说话人标注
        speaker = item.get("speaker")
        if speaker:
            text = f"[{speaker}]: {text}"

        srt_lines.append(f"{i}\n{format_time(start)} --> {format_time(end)}\n{text}\n")
```

生成的 SRT 会经过 `html.unescape()` 解码 HTML 实体。

## 兼容 OpenAI 协议的第三方服务

任何实现了 `/v1/audio/transcriptions` 端点的服务均可接入，只需修改 `base_url` 和 `api_key`：

| 服务商 | base_url | 备注 |
|--------|----------|------|
| **OpenAI** | `https://api.openai.com/v1` | 官方服务 |
| **Groq** | `https://api.groq.com/openai/v1` | 超低延迟，支持 whisper-large-v3 |
| **DeepInfra** | `https://api.deepinfra.com/v1/openai` | 支持多种开源 Whisper 模型 |
| **Together AI** | `https://api.together.xyz/v1` | 高并发支持 |
| **本地 Ollama** | `http://localhost:11434/v1` | 完全离线，需自行部署 Whisper 模型 |

## 代理连接

当全局代理启用且 `use_network_proxy = true` 时，API 请求会通过配置的 HTTP/SOCKS5 代理：

```python
if actual_use_proxy:
    client_params["http_client"] = httpx.Client(proxy=proxy_url, timeout=timeout_config)
else:
    client_params["http_client"] = httpx.Client(proxy=None, trust_env=False, timeout=timeout_config)
```

`trust_env=False` 确保非代理模式不会意外通过系统环境变量 `HTTP_PROXY` 连接。

## 错误处理

`run_api_transcription()` 实现了三级精细化错误分类：

```python
except Exception as e:
    # 1. API 业务错误 (401, 429, 500...)
    if isinstance(e, APIStatusError):
        error_message = f"❌ 云端 API 返回状态错误 (HTTP Status: {e.status_code})！\n"
        error_message += f"   - 错误详情 (Response): {e.response.text}\n"
        raise Exception(error_message)

    # 2. 网络连接错误 (DNS, Timeout, Refused...)
    elif isinstance(e, APIConnectionError):
        error_message = f"❌ 无法连接到云端 API 服务器！请检查网络或代理设置。\n"
        error_message += f"   - 错误根源: {e.__cause__}\n"
        raise Exception(error_message)

    # 3. 所有其他未知错误 — 打印完整堆栈
    else:
        traceback.print_exc()
        raise
```

| 错误类型 | 触发条件 | 前端显示 |
|----------|----------|----------|
| `APIStatusError` | HTTP 401/429/500 等 | 显示状态码和响应体 |
| `APIConnectionError` | DNS 失败/连接超时/拒连 | 提示检查网络/代理 |
| 其他 | 任意未预期错误 | 完整堆栈跟踪 |

### HTTP 客户端资源释放

无论成功或失败，自定义 `httpx.Client` 都会在 `finally` 块中强制关闭：

```python
finally:
    try:
        client_params["http_client"].close()
    except Exception:
        pass
```

## SRT 格式化

时间戳通过 `format_time()` 函数转换为标准 SRT 格式 `HH:MM:SS,mmm`：

```python
def format_time(seconds: float) -> str:
    total_milliseconds = round(seconds * 1000)
    hours = total_milliseconds // 3600000
    remainder = total_milliseconds % 3600000
    minutes = remainder // 60000
    remainder %= 60000
    secs = remainder // 1000
    msecs = remainder % 1000
    return f"{hours:02d}:{minutes:02d}:{secs:02d},{msecs:03d}"
```

## 与本地 Whisper 的对比

| 维度 | 本地 Whisper | 云端 API |
|------|-------------|----------|
| 网络依赖 | 仅首次下载模型 | 每次调用均需网络 |
| GPU 要求 | 推荐 GPU | 无需本地 GPU |
| 费用 | 免费 | 按 token/时长计费 |
| 延迟 | 取决于本地算力 | 取决于网络+服务商 |
| 精度 | faster-whisper 同等 | 取决于服务商模型 |
| 隐私 | 数据不出本机 | 音频上传至第三方 |
| 最大时长 | 无限制 | 受 API 限制 (通常 25MB/文件) |
| 说话人标注 | ❌ | ✅ (需 API 支持) |

## 故障排除

<details>
<summary>Q: 返回 401 Unauthorized？</summary>

1. 检查 API Key 是否正确（包括是否有多余空格）
2. 检查 `base_url` 是否正确
3. 确认 API Key 有对应服务的调用权限
</details>

<details>
<summary>Q: 连接超时？</summary>

1. 增加 `timeout_settings.read`（大文件可能需要数分钟）
2. 检查是否需要配置代理
3. 确认 `base_url` 可访问：`curl -I {base_url}/models`
</details>

<details>
<summary>Q: 切片后字幕时间错位？</summary>

1. 确认 `get_audio_duration()` 正常工作（FFprobe 可用）
2. 检查切片文件是否完整生成
3. 如问题持续，可手动调小 `chunk_length_sec` 增加切片密度
</details>
