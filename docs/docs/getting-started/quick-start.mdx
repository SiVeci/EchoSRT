# ⚡ 快速上手

本指南将在 5 分钟内带你完成 EchoSRT 的首次完整字幕生成流程。

---

## 第一步：启动服务

确保服务已启动并访问 Web UI：

```bash
# Docker (推荐)
docker compose up -d

# 或源码启动
python app.py
```

浏览器打开 `http://localhost:9000`，你将看到如下界面结构：

```
┌──────────────────────────────────────────────────────┐
│  Header Bar: [EchoSRT] [⚙ 全局设置] [📡 连接状态]   │
├───────────┬──────────────────┬───────────────────────┤
│           │                  │                       │
│  Tab 导航  │    操作区域       │    全局控制台          │
│ (4 个标签) │  (上传/配置)      │   (日志 + 下载)       │
│           │                  │                       │
└───────────┴──────────────────┴───────────────────────┘
```

## 第二步：上传视频

1. 点击左侧 **📂 任务工作区** 标签
2. 点击"上传视频"按钮，选择一个视频文件（支持 `.mp4`, `.mkv`, `.mov`, `.avi` 等）
3. 上传完成后，系统会自动为这个任务生成唯一的 `task_id`

> 💡 **提示**：你也可以直接上传 `.wav` 音频文件，跳过音频提取步骤。系统会对非标准格式的音频自动调用 FFmpeg 重采样为 16kHz/mono WAV。

## 第三步：配置语音识别

点击 **🎤 语音识别** 标签，选择引擎：

### 选项 A：本地 Whisper（离线识别）

1. 引擎下拉框选择 **本地识别**
2. 在模型列表中选择一个模型（首次使用会从 Hugging Face 自动下载）：
   - **large-v2** — 精度最高，需要 ~6GB 显存
   - **medium** — 平衡选择，~3GB 显存
   - **small** — 速度优先，~1GB 显存
   - **distil-large-v2** — 蒸馏模型，速度提升 ~3×
3. 语言设为 **Auto**（自动检测），或手动指定目标语言
4. VAD 语音端点检测默认启用（推荐保留）

### 选项 B：云端 API（OpenAI 兼容）

1. 引擎下拉框选择 **云端 API**
2. 填写你的 API Key（支持 OpenAI / Groq / SiliconFlow 等兼容厂商）
3. Base URL 默认 `https://api.openai.com/v1`，可改为第三方地址
4. 选择模型（`whisper-1` 或兼容模型）
5. 根据需要勾选"说话人标记"或"词级时间戳"

## 第四步：配置翻译（可选）

如果你想生成双语/翻译字幕，点击 **🌍 LLM 翻译** 标签：

1. 填写大模型 API Key
2. 选择目标语言（默认：中文 `zh`）
3. 调整并发参数：
   - **批量大小**（batch_size）：默认 50 条/批
   - **并发数**（concurrent_workers）：默认 3，越大越快但可能触发 API 限流
4. 可选：自定义系统提示词以控制翻译风格

## 第五步：启动工作流

1. 回到 **📂 任务工作区** 标签
2. 点击 **🚀 全量工作流** 按钮（提取音频 → 识别 → 翻译）
   - 如果你只需要生成原文字幕，点击 **🎤 仅提取/识别**
3. 观察右侧进度条和实时日志输出：

```
[12:30:01] 🚀 启动工作流，执行链路: [ extract ➡️ transcribe ➡️ translate ]
[12:30:02] 🎬 正在提取音频... (使用 FFmpeg)
[12:30:05] [*] 正在加载 Whisper 模型 (large-v2) 到 CUDA (float16)...
[12:30:15] [*] 开始识别音频...
[12:30:45] [*] 识别到语言: en (概率: 0.98)
[12:30:46] 🎉 提取工作流执行完毕！原声字幕已生成。
[12:30:46] 📊 共解析到 240 条字幕，准备分批请求模型...
[12:30:50] ⚡ 正在全速并发翻译中... (已完成 3/5 批)
[12:31:02] 🎉 翻译完毕！共成功处理 240/240 条字幕。
[12:31:02] 🎉 全量工作流完美收官！
```

## 第六步：下载产物

工作流完成后，右侧 **全局控制台** 区域会出现下载按钮：

| 产物 | 文件名格式 | 说明 |
|------|-----------|------|
| 📥 原文字幕 | `{文件名}.srt` | 标准 SRT 格式 |
| 📥 翻译字幕 | `{文件名}_zh.srt` | 翻译后的 SRT 字幕 |
| 📥 提取音频 | `{文件名}.wav` | 16kHz/mono WAV 音频 |

点击对应按钮即可下载到本地。

## 高级技巧

### 仅提取特定时间段的音频

在 **🎵 音频提取** 标签中设置：

```
起始时间: 00:05:00
结束时间: 00:10:00
音轨选择: 0:a:0  (选择第一个音频流)
```

这将仅提取视频第 5~10 分钟的音频，大幅减少处理时间。

### 断线重连保护

如果浏览器刷新或网络断开，系统会自动恢复正在执行中的任务：

1. 前端在 `localStorage` 中持久化当前 `task_id`
2. 页面加载时通过 `restoreActiveTask()` 探测后端状态
3. 如果后端仍在处理，自动重建 WebSocket 连接，无缝接管进度流

### 批量工作空间管理

你可以在 **任务工作区** 中创建多个并行任务，通过拖拽排序任务卡片。每个任务有独立的资产目录和运行时状态，互不干扰。
