# 📝 更新日志

本页记录 EchoSRT 各版本的显著变更，数据同步自 GitHub Releases。

---

## v1.1.0 (2026-05-10)

### ⚠️ 升级注意事项
- **配置备份**：底层配置结构已重构，升级前请务必手动备份 `config/config.json` 文件以防数据丢失。
- **缓存清理**：为确保界面正常渲染，升级后请使用 `Ctrl + F5` 强制刷新浏览器缓存。

### 🚀 API 配置管理升级
- **多方案管理池**：在翻译与云端识别页新增 API 配置方案池，支持多套凭证的预设、一键切换、克隆与安全删除。
- **配置平滑迁移**：自动将旧版 API 配置迁移至新建的“默认方案”中，且命令行 (CLI) 模式已全量适配新版方案解析逻辑。

### ⚡ 媒体库扫描优化
- **格式扩容与性能提速**：新增对 `.wmv`、`.ts`、`.flac` 等格式的支持；重构底层为单次全目录遍历算法，大幅提升超大容量媒体库的扫描速度。
- **文件状态自愈**：废弃内存状态标记，改为实时穿透工作区探测；手动清理的历史任务源文件将在下次扫描时自动恢复为可用状态。

### 🎨 系统日志与交互体验
- **后端日志静默**：屏蔽本地引擎在终端的高频滚动日志，保持后端清爽，实时日志仅在 Web 页面终端显示。
- **状态展示优化**：降低前端状态轮询负载，优化页面顶部的任务焦点追踪横幅，采用真实视频文件名替代原有的 UUID。

### 📥 资产下载机制重构
- **统一下载链路**：重构全局资产下载行为，全面唤起浏览器原生另存为窗口，并支持实时下载进度与速度反馈。
- **产物精准命名**：通过捕获真实的 `Content-Disposition` 响应头，消除因试探性请求导致的翻译产物命名错乱问题。

### 🛠️ 核心问题修复
- **资产防呆修复**：修复因虚拟映射文件导致的资产统计异常错误，并新增原生素材防误删安全锁。
- **配置覆盖保护**：确立后端文件为唯一数据源，阻断因前端网络波动或组件异常导致的关键配置覆盖风险。
- **跨平台路径指纹**：引入系统层级的感知机制，完美解决 Linux/macOS 环境下因大小写敏感导致的文件指纹去重异常。

## v1.0.1 (2026-05-08)

### 🚀 新功能与改进
- **文档中心**：集成 Docusaurus 知识库（即本 Wiki）并添加 WebUI 入口。
- **安全性增强**：实现文件名路径净化及 LLM 返回值拦截校验，提升系统健壮性。
- **性能优化**：重构 FFmpeg 进程管理机制，支持自动清理临时文件。
- **代码净化**：移除冗余代码，补全缺失的依赖声明。

## v1.0.0 (2026-05-07)

### 🏗️ 架构重构
- **内存优化**：针对低内存设备进行架构重写，通过后端模块化和异步并发队列彻底解决 OOM 问题。
- **物理切片**：引入基于 FFmpeg 的物理切片技术，提升超大文件处理能力。

### ✨ 新特性
- **代理支持**：增加全局代理配置。
- **流量控制**：实现 LLM 翻译流控制及本地模型生命周期管理。
- **用户体验**：支持任务拖拽排序，优化 Logo/Favicon 视觉展示，修复 Docker 部署问题。
