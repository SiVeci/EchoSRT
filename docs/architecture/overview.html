🏗️ 架构总览

EchoSRT 采用单进程异步架构，以 FastAPI 为 Web 框架，通过 asyncio.Queue 驱动三级流水线，前端使用 Vue 3 (基于 CDN 全局引入的零构建方案) 和 Element Plus 构建，通过 WebSocket 全双工通信接收实时进度。

层级职责​

层级路径职责
路由层api/routers/接收 HTTP 请求，参数校验，调用服务层
服务层api/services/业务逻辑编排：配置持久化、任务分发、工作区操作
状态层api/state.py全局共享数据结构：三个 asyncio.Queue + 两个 Dict
Worker 层api/workers/守护协程，消费队列，驱动 core/ 引擎
核心引擎core/纯 Python 模块：FFmpeg、faster-whisper、OpenAI API 翻译
前端frontend/基于 Vue 3 + Element Plus (CDN) 零构建方案，通过 WebSocket 全双工通信

关键设计决策​

1. 单进程多协程 vs 多进程

EchoSRT 选择 单进程 + asyncio 协程 而非 Celery/RQ 等多进程方案：

  • CPU 密集操作（Whisper 推理）通过 loop.run_in_executor() 跑在线程池中，释放 GIL 后 Python 线程可并行（C 扩展）
  • I/O 密集操作（LLM API 调用）天生适合 async/await
  • 部署简单：一条 python app.py 即可，无需额外的 broker/worker 进程
  • 配置热加载：全局 Dict 共享状态，无需序列化到 Redis
2. 队列驱动的流水线

三个 asyncio.Queue 构成无锁生产者-消费者模型：

# api/state.py
q_extract = asyncio.Queue()      # extract worker → transcribe worker
q_transcribe = asyncio.Queue()   # transcribe worker → translate worker
q_translate = asyncio.Queue()    # translate worker → 完成

每个 Worker 是独立的守护协程，通过 queue.task_done() 标记完成，支持优雅关闭。

3. 基于 Vue 3 的前端无构建方案

EchoSRT 的前端采用 Vue 3 和 Element Plus 构建，通过 CDN 全局引入，实现了原生无构建方案。

好处：极简部署、免安装 Node.js、开箱即用，通过 app.py 直接作为静态资源伺服。

4. 静态文件服务与前端挂载

app.py 中通过 Mount 将 / 路径挂载到 frontend/ 目录：

app.mount("/", StaticFiles(directory="frontend", html=True), name="frontend")

html=True 使 404 请求回退到 index.html，支持前端单页应用。

启动模式​

EchoSRT 支持两种运行模式，共享 core/ 引擎层：

  1. Web 模式 (app.py)：启动 FastAPI 服务与后台 Worker 管道，支持多任务并发与 Web UI。
  2. CLI 模式 (main.py)：轻量级同步流工具，适合本地单任务快速处理，不启用队列和异步 Worker。
app.py ──启动──▶ uvicorn(app, port=8000)
    │
    ├── 1. 初始化 workspace/ 目录
    ├── 2. 加载 config.json（若存在）
    ├── 3. 启动 Worker 守护协程 × 3（后台任务车间）
    └── 4. 开始监听 HTTP 请求

详见 流水线引擎 和 状态管理。
