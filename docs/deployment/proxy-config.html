🌐 代理与网络配置

在国内网络环境下，HuggingFace 模型下载和 LLM API 调用可能受阻。EchoSRT 内置了系统级 HTTP/SOCKS5 代理支持。

代理配置入口​

通过 EchoSRT 前端的 全局设置 面板或直接编辑 config/config.json：

{
  "system_settings": {
    "enable_global_proxy": true,
    "network_proxy": "http://192.168.1.100:7890"
  }
}

支持的代理协议​

协议示例说明
HTTPhttp://127.0.0.1:7890最通用
HTTPShttps://proxy.example.com:8443加密代理
SOCKS5socks5://127.0.0.1:1080自动转为 socks5h://
SOCKS5Hsocks5h://127.0.0.1:1080DNS 也走代理

实现原理与直连模式​

EchoSRT 的代理架构设计允许对不同的外部请求提供细粒度的网络控制（由 system_settings.enable_global_proxy 作为总开关）：

  1. 环境变量注入：api/services/config_service.py 会在后端启动或修改配置时动态设定 HTTP_PROXY / HTTPS_PROXY 环境变量。
  2. socks5 → socks5h 转化：强制所有 SOCKS5 代理在远端执行 DNS 解析，防止本地 DNS 被污染。
  3. 显式直连穿透 (trust_env=False)：为了防止全局环境变量误伤特定模块，核心服务在初始化 httpx.AsyncClient 等 HTTP 客户端时，如果检测到用户未针对该模块开启代理，会强制传入 proxy=None, trust_env=False，彻底无视环境变量，实现真正的物理直连。

Docker 容器中启用代理​

方法 1：通过前端 GUI​

  1. 访问 EchoSRT Web 界面
  2. 点击右上角 全局设置
  3. 启用 全局代理，填入代理地址（如 http://host.docker.internal:7890）
  4. 保存后，新启动的任务自动应用代理

方法 2：docker-compose 环境变量​

services:
  echosrt-cpu:
    environment:
      - HTTP_PROXY=http://192.168.1.100:7890
      - HTTPS_PROXY=http://192.168.1.100:7890

方法 3：config.json 预配置​

{
  "system_settings": {
    "enable_global_proxy": true,
    "network_proxy": "http://192.168.1.100:7890"
  }
}

将 config.json 放入挂载的 config/ 目录。

常见代理场景​

Clash / Clash Verge​

{
  "network_proxy": "http://127.0.0.1:7890"
}

V2Ray / Xray​

{
  "network_proxy": "socks5://127.0.0.1:10808"
}

旁路由 / OpenWrt​

{
  "network_proxy": "http://192.168.1.1:8080"
}

Docker Desktop (macOS/Windows)​

{
  "network_proxy": "http://host.docker.internal:7890"
}

host.docker.internal 是 Docker Desktop 提供的特殊 DNS，指向宿主机。

代理颗粒度配置映射​

每个组件都有独立的代理开关（需要全局 enable_global_proxy=true 且配置了 network_proxy 才会真正生效）：

场景config.json 对应 Key直连表现
下载 faster-whisper 模型 (HuggingFace)system_settings.use_proxy_for_model_download显式传入 proxies={"http": None} 覆盖
在线 ASR API 调用online_asr_settings.use_network_proxy传入 trust_env=False 实现隔离直连
LLM 翻译 API 调用llm_settings.use_network_proxy传入 trust_env=False 实现隔离直连
内部服务通信无需配置NO_PROXY=localhost,127.0.0.1 硬编码隔离

故障排查​

Q: 代理设置后模型依然无法下载？
  1. 确认代理服务在宿主机上已启动并监听
  2. 测试代理连通性（在容器内）： 
    docker exec echosrt-cpu curl -x http://PROXY_IP:PORT https://huggingface.co
  3. 检查防火墙是否开放了代理端口
  4. 查看日志确认代理已生效：docker logs echosrt-cpu | grep "已启用全局网络代理"
Q: 设置代理后本地前端无法访问？

NO_PROXY=localhost,127.0.0.1,::1 已排除本地地址。如果仍受影响，检查代理工具是否设置了全局系统代理（TUN 模式），可能需要调整代理规则排除 127.0.0.1。

Q: SOCKS5 代理无法解析域名？

EchoSRT 自动将 socks5:// 改写为 socks5h://。但某些代理客户端不支持 socks5h 变体。解决：改用 HTTP 代理，或在代理客户端开启 UDP/TCP 转发。
