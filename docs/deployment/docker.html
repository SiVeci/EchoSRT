🐳 Docker 部署

EchoSRT 提供预构建的 Docker 镜像，支持一键启动 CPU 或 GPU 版本，适配各类 NAS 和服务器环境。

前置要求​

依赖最低版本用途
Docker20.10+容器运行时
Docker Composev2 (plugin) 或 1.27+多容器编排
FFmpeg内置在镜像中无需单独安装

快速启动​

CPU 版本​

# 1. 创建项目目录
mkdir echosrt && cd echosrt

# 2. 下载 docker-compose.yml
wget https://raw.githubusercontent.com/SiVeci/EchoSRT/main/docker-compose.yml

# 3. 启动 CPU 服务
docker compose up -d echosrt-cpu

启动后访问 http://localhost:8000。

GPU 版本​

# 前提：安装 NVIDIA Container Toolkit
# 详见 gpu-setup 文档

# 启动 GPU 服务（默认映射 8001 端口，避免和 CPU 版冲突）
docker compose up -d echosrt-gpu

环境变量​

变量默认值说明
PUID0文件所有者 UID，NAS 环境建议设为非 root
PGID0文件所有者 GID
TZUTC时区，如 Asia/Shanghai
SKIP_CHOWNfalse设为 true 跳过权限修改（加速启动）

卷挂载 (Volumes)​

volumes:
  - ./workspace:/app/workspace   # 必需：任务产物目录
  - ./models:/app/models         # 必需：下载的 Whisper 模型持久化
  # - ./config:/app/config       # 可选：持久化用户配置

挂载策略​

目录内容作用
workspace/每个任务的音频、SRT 文件保持任务结果不因容器重启丢失
models/faster-whisper 下载的模型文件 (几百 MB ~ 几 GB)避免每次重启重新下载
config/config.json + config.example.json跨容器保留自定义配置

注意: 模型文件极大，首次启动时下载可能需要数分钟。建议将 models/ 目录持久化挂载。

资源限制​

Docker Compose 中内置了内存限制，防止小内存 NAS 因为下载大模型导致系统卡死：

deploy:
  resources:
    limits:
      memory: 4G          # 限制最大内存

可根据你的设备内存调整此值。

CPU 版本 Dockerfile 详解​

# Dockerfile
FROM python:3.10-slim           # 轻量 Python 镜像 (~150 MB)
RUN apt-get install ffmpeg gosu  # 安装 FFmpeg + 权限工具
RUN pip install -r requirements.txt
COPY . .
ENTRYPOINT ["/app/docker-entrypoint.sh"]
CMD ["uvicorn", "app:app", "--host", "0.0.0.0", "--port", "8000"]

关键设计​

  1. gosu 降权: entrypoint 根据 PUID/PGID 创建非 root 用户，以普通用户身份运行 uvicorn
  2. 浅层 chown: 使用 find -maxdepth 1 而非递归 chown，防止巨量模型碎片文件引发启动假死
  3. 配置兜底: 将 config.example.json 备份到 /app/ 根目录；如果挂载的 config 目录为空，自动恢复

GPU 版本 Dockerfile 详解​

# Dockerfile.gpu
FROM nvidia/cuda:12.2.2-cudnn8-runtime-ubuntu22.04  # NVIDIA CUDA 基础镜像 (~3 GB)
ENV LD_LIBRARY_PATH=/usr/local/cuda/lib64:...        # CTranslate2 自动检测 CUDA
RUN apt-get install python3.10 ffmpeg gosu
RUN pip install -r requirements.txt

GPU 镜像基于 NVIDIA 官方 CUDA 镜像构建，faster-whisper 底层的 CTranslate2 库会自动检测并启用 GPU 推理，无需额外配置。

自定义构建​

如需本地构建镜像（而非拉取 GitHub 预编译镜像）：

# docker-compose.yml 中取消注释 build 并注释 image
echosrt-cpu:
  # image: ghcr.io/siveci/echosrt:cpu-latest
  build:
    context: .
    dockerfile: Dockerfile
docker compose build --no-cache echosrt-cpu
docker compose up -d echosrt-cpu

常用运维命令​

# 查看日志
docker logs -f echosrt-cpu

# 进入容器
docker exec -it echosrt-cpu bash

# 重启服务
docker compose restart echosrt-cpu

# 停止并清理
docker compose down

# 更新镜像（拉取最新版）
docker compose pull echosrt-cpu
docker compose up -d echosrt-cpu

网络说明​

默认监听 0.0.0.0:8000（CPU 版）和 0.0.0.0:8001（GPU 版）。通过 ports 映射可更改主机端口。EchoSRT 为纯 HTTP/WS 服务，不涉及 HTTPS/TLS（建议使用 Nginx/Caddy 反向代理）。
