# <img src="icons/folder.svg" width="20" height="20" style="vertical-align:-3px"> 媒体库扫描

媒体库功能允许 EchoSRT 深度扫描本地磁盘或 NAS 挂载的文件夹，自动识别视频文件并支持批量导入为工作流任务。这对于拥有大量存量视频素材的用户非常实用。

---

## 工作流程

1. **添加路径**：在“全局设置”中指定一个或多个本地目录。
2. **执行扫描**：点击工作区的“扫描媒体库”按钮，系统将递归遍历目录。
3. **勾选导入**：从扫描结果弹窗中选择需要处理的文件。
4. **一键分发**：导入后的文件自动出现在任务列表，可进行批量提取、识别和翻译。

---

## 配置指南

### 1. 添加扫描路径
进入 **全局设置** -> **媒体库扫描管理**：
- 输入绝对路径（如 Windows 的 `D:\Media` 或 Linux 的 `/mnt/videos`）。
- 点击“添加路径”。系统会即时校验路径是否存在且可读。

### 2. 执行扫描
在 **工作区看板** 顶部点击 **<img src="icons/search.svg" width="16" height="16" style="vertical-align:-2px"> 扫描媒体库**。
- 扫描是异步进行的，完成后会弹出“媒体库扫描发现”对话框。
- 系统会根据文件指纹（Fingerprint）自动过滤掉已经导入过的文件，避免重复导入。
- **任务持久化**：导入后的任务状态会实时保存至 `workspace/{task_id}/state.json`，即使重启后端服务，扫描导入的任务进度也会被保留。

## <img src="icons/box.svg" width="20" height="20" style="vertical-align:-3px"> Docker 与 NAS 特别说明

如果你在 Docker 容器（如群晖、Unraid 或威联通）中运行 EchoSRT，必须将宿主机的媒体目录**挂载**到容器内，否则后端无法访问。

### Docker Compose 配置示例
```yaml
services:
  echosrt-cpu:
    volumes:
      - /volume1/video:/mnt/media:ro  # 将 NAS 视频目录挂载为只读
```

### 添加路径时注意
在 EchoSRT 的 Web 界面中添加路径时，应填入**容器内部的路径**（即上方示例中的 `/mnt/media`），而非宿主机的原始路径。

## 常见问题

<details>
<summary><b>Q: 扫描不到视频文件？</b></summary>

1. 检查 `config/config.json` 中的 `allowed_extensions` 是否包含该文件的后缀。
2. 确认 Docker 挂载路径是否正确。
3. 确认文件夹权限是否允许运行 EchoSRT 的用户读取。
</details>

<details>
<summary><b>Q: 导入后文件去哪了？</b></summary>

导入操作不会移动或复制原始视频文件。它会在 `workspace/` 下创建一个虚拟任务，并将其 `absolute_path` 指向原始位置。因此：
- 节省磁盘空间：不产生视频副本。
- 安全性：删除任务仅删除生成的 SRT 和音频，不会触碰你的原始视频。
</details>
