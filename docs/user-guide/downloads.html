结果下载与文件管理

EchoSRT 处理完成后生成的字幕文件可通过 Web UI 直接下载，结构清晰，命名规范。

产物文件结构​

每次任务完成后，在 workspace/{task_id}/ 目录下生成以下文件：

workspace/<task_id>/
├── video.mp4          # 用户上传的原始视频文件
├── audio.wav          # 提取或标准化重采样的音频（16kHz/mono）
├── original.srt       # Whisper 生成的原始语言字幕
└── translated.srt     # LLM 生成的目标语言字幕

文件说明​

文件阶段说明
video.*用户上传保留原始扩展名，如果重复上传将覆盖旧文件
audio.wav音频提取由视频提取或由独立音频经 FFmpeg 重采样生成
original.srt语音识别标准 SRT 格式，源语言字幕
translated.srtLLM 翻译标准 SRT 格式，目标语言字幕

下载产物与命名规则​

API 下载​

下载时由 get_download_file_path 动态计算安全的文件名，防止路径穿越，并附加元数据中的 base_name 保持语义可读性：

GET /api/tasks/{task_id}/download/{type}

支持的 type 及命名规则：

  • original: 返回 original.srt，下载名为 {base_name}.srt
  • translated: 返回 translated.srt，下载名为 {base_name}_{lang_code}.srt
  • audio: 返回 audio.wav，下载名为 {base_name}.wav
  • video: 返回 video.*，下载名为 {base_name}{ext}

文件命名规则​

原始字幕 (original.srt)​

1
00:00:00,000 --> 00:00:03,500
这是识别的第一段文字。

2
00:00:03,500 --> 00:00:07,200
这是识别的第二段文字。

翻译字幕 (translated.srt)​

时间轴与原始字幕完全对齐：

1
00:00:00,000 --> 00:00:03,500
This is the first recognized text.

2
00:00:03,500 --> 00:00:07,200
This is the second recognized text.

资产清理​

与 workspace_service.py 的底层逻辑对齐，资产清理具备严格的并发与安全限制：

方式规则说明
单资产删除删除 video/audio/original/translated 中的任一项（为防止任务损坏，系统强制要求至少保留一份有效资产）
整任务清理递归删除整个 {task_id} 目录，同时清理相关的持久化记录与 WebSocket 状态
并发保护当任务正在 extracting、transcribing、translating 等处理阶段时，系统将阻断所有删除请求
